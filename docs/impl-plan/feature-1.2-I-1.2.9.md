# I-1.2.9: Event Image Upload

**Feature ID:** I-1.2.9  
**Module:** 1.2 — Event Creation & Management  
**Status:** ✅ Complete (2026-04-27)  
**Dependencies:** I-1.2.1 — Event creation form; I-0.1.8 — Object storage client  
**Downstream:** I-2.1.1 — Professional event page layout

## Scope

Allow organizers to upload and manage the event hero image and route map through S3/R2 presigned PUT URLs, with server-side validation and persisted image metadata.

## Acceptance Criteria

1. Shared constants and Zod schemas define image kinds, statuses, content types, file size limits, upload request/response payloads, and persisted response shapes.
2. Database schema includes `event_images` with event/user foreign keys, storage key uniqueness, content type checks, active image indexing, and migration/rollback coverage.
3. API exposes organizer image upload URL, confirm, list, and delete endpoints under `/api/v1/events/:eventId/images`.
4. Upload keys are server-generated, image mutations require organizer auth/ownership, confirm validates object metadata, and replacement marks previous active images as replaced.
5. Web exposes server functions/query options and `/org/events/$eventId/configure-images` UI for hero image and route map uploads.

## Key Endpoints and Routes

- `POST /api/v1/events/:eventId/images/upload-url` — request presigned S3/R2 PUT URL.
- `POST /api/v1/events/:eventId/images/:imageId/confirm` — confirm uploaded object metadata and mark image uploaded.
- `GET /api/v1/events/:eventId/images` — list owner-visible pending/uploaded images or public uploaded images.
- `DELETE /api/v1/events/:eventId/images/:imageId` — delete storage object and mark image deleted.
- `/org/events/$eventId/configure-images` — organizer UI for hero image and route map upload/replacement.
